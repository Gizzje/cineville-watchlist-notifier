# Cineville Watchlist Notifier

A small Python script + Telegram bot that watches your [Cineville](https://cineville.nl)
watchlist and tells you when a film that hasn't been playing yet or for a
while shows up again in the weekly cinema schedule.

## Why

Cineville (a Dutch "unlimited cinema" subscription) lets you keep a
watchlist and filter the weekly schedule by it, but it has **no
notifications** and **no official API**. This project fills that gap: it
queries the same (undocumented) API cineville.nl's own website uses,
compares this week's schedule against your watchlist and a rolling history
of recent weeks, and pings you on Telegram when something new shows up.

## Features

- Compares your Cineville watchlist against the weekly agenda for one or
  more cities.
- Only notifies about films that were *absent* from the schedule for a
  configurable number of weeks (default 4) before reappearing — so a film
  that's been running continuously won't spam you every week.
- Adds a clickable link to the film's Cineville page (in Dutch or English,
  depending on your language setting).
- Optionally adds IMDb / Rotten Tomatoes ratings via [OMDb](https://www.omdbapi.com/).
- Weekly heartbeat message so you know the check is still running.
- Proactively warns you before your Cineville login session expires.
- Notifications in Dutch or English, switchable via config or a bot command.
- Optional Telegram bot listener so you can change settings
  (`/setweeks`, `/setcookie`, `/setlanguage`, `/checknow`, `/status`)
  without touching the server.

## How it works

Cineville's weekly schedule runs **Thursday through Wednesday**, and the
next week's schedule is published Monday evening. The script:

1. Exchanges your stored Cineville session cookie for a fresh API access
   token.
2. Fetches your watchlist.
3. Fetches the upcoming Thursday–Wednesday schedule for the configured
   cities.
4. Compares the two, and checks a local `history.json` to see which matches
   are genuinely new (absent for `lookback_weeks` weeks).
5. Sends a Telegram message for each new match, plus one heartbeat summary.

There is no official Cineville API — the endpoints used here were found by
reading cineville.nl's own (minified) frontend code. They could change or
break at any time; see [SETUP.md](SETUP.md) for how to diagnose that.

## Requirements

- A Cineville account with a filled-in watchlist.
- Python 3.10+ on any machine that can run a weekly cron job (a Raspberry
  Pi, a VPS, a Proxmox LXC container, your own laptop with `cron`/Task
  Scheduler, ...).
- A Telegram bot (free, see [SETUP.md](SETUP.md)).
- Optional: a free [OMDb](https://www.omdbapi.com/apikey.aspx) API key for
  ratings.

## Quick start

See **[SETUP.md](SETUP.md)** for the full step-by-step walkthrough
(Telegram bot, extracting your Cineville session cookie, configuration,
deployment, and the cron job).

```bash
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
cp config.example.json config.json   # then fill it in, see SETUP.md
./venv/bin/python cineville_notify.py config.json
```

## Configuration reference (`config.json`)

| Field                | Required | Description                                                                 |
|----------------------|----------|-------------------------------------------------------------------------------|
| `cineville_cookie`   | yes      | Raw `Cookie` header from a logged-in cineville.nl session (see SETUP.md).      |
| `telegram_bot_token` | yes      | Token from [@BotFather](https://t.me/BotFather).                              |
| `telegram_chat_id`   | yes      | Your Telegram chat ID (e.g. via [@userinfobot](https://t.me/userinfobot)).     |
| `cities`             | yes      | List of Cineville city slugs to check, e.g. `["amsterdam", "amstelveen"]`.     |
| `lookback_weeks`     | no       | Weeks a film must be absent before a reappearance counts as "new" (default 4). |
| `history_file`       | no       | Where weekly schedule state is stored (default `history.json`).               |
| `bot_offset_file`    | no       | Used by the bot listener to track processed Telegram messages.                |
| `omdb_api_key`       | no       | Free OMDb key for IMDb/Rotten Tomatoes ratings; omit to skip ratings.          |
| `cookie_warning_days`| no       | Days before session expiry to start warning you (default 5).                  |
| `language`           | no       | `"nl"` (default) or `"en"` — language of all Telegram messages and links.     |

## Limitations

- Uses an **undocumented** Cineville API — it works today, but Cineville
  could change it at any time.
- Your Cineville login session needs to be refreshed manually every so
  often (the script warns you before it expires — see SETUP.md for how
  long that typically lasts and how to grab a fresh one).
- Only Dutch and English are supported; other languages would need a new
  entry in the `MESSAGES` dict in `cineville_notify.py` (and `BOT_MESSAGES`
  in `cineville_bot_listener.py`).
- Only tested with Amsterdam/Amstelveen, but any valid Cineville city slug
  should work.
